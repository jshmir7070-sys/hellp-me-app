1) 2026년 제출 기준 “필수 게이트” (이거 못 맞추면 업로드 단계에서 막히는 경우가 많음)

아래는 신규 앱/업데이트 제출 자체가 막히거나, 검수에서 고확률로 반려되는 항목입니다.

A. Target API(타겟 SDK) 충족

2025-08-31부터 신규 앱 + 업데이트는 Android 15(API 35) 이상을 타겟해야 제출 가능합니다.
→ 2026-01-30 현재는 사실상 targetSdkVersion 35+가 기본 전제입니다.

B. AAB(App Bundle)로 배포

신규 앱은 APK가 아니라 Android App Bundle(AAB) 로 게시해야 합니다.

C. 64-bit 지원

Google Play 배포 앱은 64-bit 아키텍처 지원이 필요합니다(대표적으로 arm64-v8a).

D. 16 KB Page Size 지원 (특히 target=35면 사실상 필수로 봐야 함)

2025-11-01부터 “Android 15+ 대상(= API 35 타겟)” 신규/업데이트 앱은 16KB page size를 지원해야 합니다.

Android 15부터 16KB 페이지 사이즈 기기가 등장했고, 앱이 NDK 네이티브 라이브러리(.so)를 직접/간접(서드파티 SDK 포함)으로 쓰면 재빌드/호환이 필요합니다.

즉, 맵스토어가 API 35를 타겟하려면(현재는 그래야 제출 가능)
(1) AAB + (2) 64-bit + (3) 16KB 호환까지 한 세트로 맞춰야 “업로드”가 안정적입니다.

2) 맵/위치 기반 앱에서 자주 반려 나는 포인트
A. 백그라운드 위치 권한(ACCESS_BACKGROUND_LOCATION)

백그라운드 위치는 핵심 기능(코어 유즈케이스) 이 아니면 승인 받기 어렵고, 광고 목적만으로는 거절됩니다.

가능하면 설계를 “앱 사용 중(While-in-use) 위치” 중심으로 바꾸고, 정말 필요할 때만 백그라운드 위치를 쓰는 구조가 안전합니다.

B. 개인정보/민감정보 수집 시 “앱 내 고지+동의” (Prominent Disclosure & Consent)

개인정보/민감정보를 기능상 명확하지 않은 방식으로 수집/전송한다면, 앱 내부에서 눈에 띄는 고지와 동의가 요구됩니다.

C. Data safety(데이터 안전) + 개인정보처리방침

Play Console의 Data safety 섹션을 채우려면 개인정보처리방침(Privacy Policy) URL이 사실상 필수입니다.

D. 계정 생성이 있다면 “계정 삭제” 요건

앱에서 계정을 만들 수 있으면 (1) 앱 내 계정 삭제 경로 + (2) 웹에서 삭제 요청 가능한 링크가 필요합니다.

E. 인앱결제(디지털 상품/구독 등)가 있다면 Billing Library v7+

인앱결제가 있다면, **2025-08-31부터는 Billing Library v7+**가 요구됩니다(신규/업데이트).

F. “앱이 너무 단순/기능 부족” 판정(최소 기능/품질)

기능이 거의 없거나(단순 웹뷰/정적 뷰어 수준 등) 크래시/동작 불량이면 제한될 수 있습니다.

3) 맵스토어 프로젝트에서 “어디를 보면” 등록 가능 여부가 바로 갈립니다

아래 파일/값만 보면 등록 가능성 70~80%는 판단됩니다.

(1) Android 타겟/빌드 설정

android/app/build.gradle 또는 android/app/build.gradle.kts

compileSdk / targetSdk → 35 이상인지

minSdk (너무 낮아도 문제라기보단 호환/보안 이슈 가능)

ndkVersion / androidGradlePlugin(AGP) 버전 (16KB/네이티브 이슈에 영향)

(2) 권한/특수 API

android/app/src/main/AndroidManifest.xml

위치: ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION, ACCESS_BACKGROUND_LOCATION 여부

파일/미디어: 과도한 저장소 접근(예: broad storage) 권한 여부(필요 시 정책 이슈 가능)

다른 고위험 권한(문자/통화기록 등) 있으면 거의 무조건 점검 필요

(3) AAB 생성 및 ABI 확인 (64-bit + 16KB 관점에서 중요)

Release AAB 생성:

./gradlew bundleRelease

생성된 .aab에 arm64-v8a 포함 여부 확인(64-bit)

만약 서드파티 SDK가 .so를 포함하면 16KB 호환이 관건(업데이트/교체 필요할 수 있음).

4) “지금 단계에서” 제가 드릴 수 있는 판단

원칙적으로 등록 가능: 일반적인 지도/위치 기반 상점 탐색 앱은 Google Play(플레이스토어) 정책상 “카테고리” 자체가 금지된 건 아닙니다.

다만 2026년 현재 제출 요건은:

API 35+

아아비

64비트

(API 35 타겟이면) 16KB page size
이 4가지를 충족해야 “기술적으로 제출 가능”에 들어갑니다.

그리고 맵스토어가 위치를 쓴다면:

백그라운드 위치를 쓰는지 여부에 따라 정책 리스크가 크게 달라집니다.

로그인/회원 기능이 있다면:

계정 삭제(앱 내 + 웹 링크) 까지 챙겨야 반려를 피하기 쉽습니다
